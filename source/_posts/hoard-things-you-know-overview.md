# 囤积你知道如何做的事情

这是 agentic engineering 的另一个模式：**囤积你知道如何做的事情**。

这里的想法很简单：每次你学习如何做某事——无论是使用新库、解决特定类型的问题，还是实现特定类型的功能——都要记录下来。写下你做了什么，你使用了什么工具，你学到了什么。

这听起来像是基本的建议，但它比以往任何时候都更重要。

## 为什么这很重要 [#](/guides/agentic-engineering-patterns/hoard-things-you-know-how-to-do/#why-this-matters)

当你使用编码 agent 时，你正在建立自己的"知道如何做"的库存。每次你解决一个新问题，你就有了可以重用的东西。

但这里有一个陷阱：如果你不记录你做了什么，你就无法有效地重用它们。

我见过很多开发人员在使用编码 agent 时陷入这个陷阱。他们解决了一个问题，几周后遇到了类似的问题，然后从头开始。

不要这样做。

## 如何做到这一点 [#](/guides/agentic-engineering-patterns/hoard-things-you-know-how-to-do/#how-to-do-this)

有几种方法可以做到这一点：

### 1. 保存你的提示

每次你使用编码 agent 时，保存你使用的提示。这包括：

* 你给 agent 的初始提示
* 任何后续提示
* 任何你用来改进结果的技巧

### 2. 保存你的代码

保存 agent 为你生成的代码。即使你现在不使用它，将来你可能会需要它。

### 3. 记录你的学习

每次你学习新东西时，记录下来。这可以是：

* 博客文章
* 笔记
* 代码注释
* 任何对你有意义的东西

## 我的方法 [#](/guides/agentic-engineering-patterns/hoard-things-you-know-how-to-do/#my-approach)

我使用几种方法来囤积我知道如何做的事情：

### 1. 我的博客

我在 [simonwillison.net](https://simonwillison.net/) 上写博客。每次我学习新东西时，我都会写一篇关于它的文章。

这有几个好处：

* 它迫使我清楚地思考我学到了什么
* 它为我创建了一个可搜索的知识库
* 它帮助其他人学习同样的东西

### 2. 我的 GitHub 仓库

我在 GitHub 上有 [77 个 HTML+JavaScript 应用程序和 6 个 Python 应用程序](https://github.com/simonw/tools)。每一个都是通过提示 LLM 构建的。

我使用这些作为学习新库和新技术的方式。

### 3. 我的笔记

我使用 [Obsidian](https://obsidian.md/) 来做笔记。每次我学习新东西时，我都会记下来。

这创建了一个我可以搜索的个人知识库。

## 开始行动 [#](/guides/agentic-engineering-patterns/hoard-things-you-know-how-to-do/#get-started)

开始囤积你知道如何做的事情。

每次你学习新东西时，记录下来。保存你的提示。保存你的代码。

将来你会很高兴你这么做了。
