# 先运行测试

在 agentic engineering 中，一个重要的模式是**先运行测试**。

## 为什么先运行测试

当你使用 AI agent 编写代码时，一个重要的实践是让 agent 先运行现有的测试，了解代码库的行为和期望。

## 如何让 agent 先运行测试

### 方法 1：在提示中明确说明

```
在开始编写代码之前，先运行现有的测试套件，了解：
1. 测试如何运行
2. 现有测试覆盖的功能
3. 代码库的编码风格
```

### 方法 2：让 agent 探索测试目录

```
先查看 tests/ 目录，了解：
1. 测试文件结构
2. 使用的测试框架
3. 测试命名约定
```

### 方法 3：让 agent 运行特定测试

```
运行与这个功能相关的测试，确保你理解：
1. 测试如何断言
2. 测试使用的 mock 方式
3. 测试的输入输出格式
```

## 好处

1. **理解代码库** - 通过测试了解代码的预期行为
2. **避免破坏现有功能** - 知道哪些测试不能破坏
3. **学习编码风格** - 了解项目的编码约定
4. **快速验证** - 知道如何验证你的改动是否正确

## 示例对话

**用户**: 帮我添加一个新功能

**Agent**: 好的，在开始之前，让我先运行现有的测试，了解代码库的结构...

```bash
$ npm test
# 运行测试，观察输出
```

现在我对代码库有了更好的理解，让我开始实现新功能...
